# Audit Report: Netflix Safari Language Learner

**Date:** 2026-03-12
**Files scanned:** 22 source files, 5 doc/test files
**Issues found:** 20 (P0: 2, P1: 6, P2: 5, P3: 7)

---

## P0 — Broken / Will Cause Bugs

| File | Line(s) | Issue | Suggested Fix |
|------|---------|-------|---------------|
| `content-script.js` | 10 | **No teardown on SPA navigation.** `bootstrapped` is set to `true` once and never resets. When user navigates `/watch/123` → `/browse` → `/watch/456`, the extension never re-bootstraps — adapter, stores, and controllers carry stale state from the previous watch session. The `checkRoute` poll runs but `bootstrapWatchRuntime()` returns immediately. | Track the current watch URL or video ID; when it changes, destroy the previous runtime and create a new one. Or reset `bootstrapped = false` when leaving `/watch`. |
| `platform/netflix-injected.js` | superseded | **Superseded.** The old content-script auto-pause controller was removed. Auto-pause is now page-owned and scheduled against the page player clock and cue timeline directly. | Keep this audit item as historical only; do not reintroduce a content-script pause-pending state. |

## P1 — Inconsistent / Race Conditions

| File | Line(s) | Issue | Suggested Fix |
|------|---------|-------|---------------|
| `core/translation-queue.js` | 195-199 | **Un-awaited recursive `flush()`.** In the `finally` block, `flush()` is called without `await`. Errors from subsequent flushes are silently swallowed. If multiple prefetch calls enqueue items while a flush is in progress, the chained flushes can pile up as untracked promises. | `await flush()` or use a `queueMicrotask` pattern with error handling. |
| `core/translation-queue.js` | 203-273 | **`prefetch()` has sequential awaits over shared mutable state.** The `for...of` loop calls `await databaseClient.getSubtitleTranslation()` for each cue. Between awaits, a settings change can trigger `clear()` (line 321-326), wiping `entries` and `pending`. The loop continues adding to the now-cleared maps, creating orphaned entries. | Check `lastTargetLanguage` consistency after each await, or batch the DB lookups. |
| `platform/netflix-adapter.js` | 500-506 | **MutationObserver on `document.documentElement` with `subtree: true` triggers `syncVideo()` on every DOM mutation.** No debounce. During Netflix player transitions (loading, buffering, UI changes), this fires dozens of times per frame. Each call does `document.querySelectorAll('video')` and potentially emits `playerReady` + `resetSubtitleState()`. | Debounce `scanDom()` with a `requestAnimationFrame` or small timeout. |
| `core/control-actions.js` | 66-76 | **`seekToTarget` resume logic appears inverted.** `shouldResume = video.paused` captures that the video IS paused. Then `if (shouldResume && video.paused) { playPlayback() }` means "if video was paused and is still paused after seek, force-play it." This would auto-play a paused video whenever the user navigates subtitles, which seems unintended for pause-and-navigate workflows. | Likely should be `const wasPlaying = !video.paused` and resume only if it was playing. |
| `content-script.js` | 1170-1192 | **Initialization chain: `settingsStore.load()` → `adapter.init()` → `syncFromAdapter()` runs without error boundary per step.** If `adapter.init()` throws, `syncFromAdapter` and `databaseClient.open()` never run. But `adapter.init()` starts a MutationObserver and message listener that may still fire, calling into uninitialized stores. | Add per-step error handling so partial init doesn't leave dangling observers. |
| `ui/control-integration.js` | 507-531, 533-556 | **`ensureMounted()` and `render()` double-install listeners.** `ensureMounted()` nulls out `cleanupPlaybackControlListeners` at line 528 but doesn't re-install them. Then `render()` calls `attachPlaybackInterception()` which installs them again. If `visibilityEnabled` is false, `detachPlaybackInterception()` runs, but there's a brief window where both visibility and playback listeners are torn down and re-installed on every render cycle. | Consolidate listener lifecycle into one place instead of split between `ensureMounted` and `render`. |

## P2 — Stale / Redundant

| File | Line(s) | Issue | Suggested Fix |
|------|---------|-------|---------------|
| `manifest.json` | 31 | **`extension-api.js` loaded twice.** Listed in both content_scripts entries (document_start and document_end). The second load overwrites `app.extensionApi`, resetting the `debugEntries` array. Any debug entries recorded between document_start and document_end are lost. | Remove from the document_end entry since `inject.js` (document_start) already loaded it, and it persists on `globalThis`. |
| `utils/language-utils.js` | 235-240 | **`toTranslationKey()` ignores `title` parameter.** Key is `targetLanguage::text` only. Different shows with identical subtitle text share cache entries. Context-dependent translations (e.g., "Yes, sir" in a period drama vs modern show) would get wrong cached results. | Include title in the key, or remove the unused parameter to avoid confusion. |
| `content-script.js` | 343, 361 | **`getTranslationLogPayload()` called twice in `saveTranslationLog()`.** Line 343 serializes the full payload. Line 361 calls it again just to count entries. | Reuse the first result: `const data = getTranslationLogPayload(); ... entryCount: data.entries.length`. |
| `core/control-actions.js` | 121-131 | **`togglePlayPause()` returns inconsistent values.** Returns `playPlayback()` result when unpausing, but returns `false` after calling `pausePlayback()`. Callers can't rely on the return value to know if the action succeeded. | Return `pausePlayback()` result, or always return void. |
| `utils/language-utils.js` | 20 | **Hardcoded model ID `grok-4-1-fast-non-reasoning-latest`** will go stale. | Reference from a single config source rather than hardcoding. |

## P3 — Quality / Unnecessary Complexity

| File | Line(s) | Issue | Suggested Fix |
|------|---------|-------|---------------|
| `content-script.js` | 67-1006 | **~940 lines of debug/trace infrastructure in production code.** `getVisibleControlsSnapshot`, `saveVisibilityTrace`, `savePlaybackTransitionTrace`, `createRollingVisibilityRecorder`, `buildVisibilityTraceRecord`, etc. The actual runtime logic is ~100 lines (1047-1207). Debug code is 9x the feature code. | Extract debug tools to a separate optional file loaded only in dev mode, or gate behind a debug flag in manifest. |
| Multiple files | — | **`traceTranslation()` defined identically in 3 files.** `subtitle-store.js:7-15`, `overlay-controller.js:8-15`, `translation-queue.js:7-15` all define the same function. | Extract to a shared utility, e.g., `utils/trace.js`. |
| Multiple files | — | **`isVisibleElement/Node` duplicated 3 times.** `content-script.js:148-160`, `overlay-controller.js:81-93`, `control-integration.js:169-181` all implement the same visibility check. | Move to `dom-utils.js` as a shared function. |
| `content-script.js` + `overlay-controller.js` | 96-134 / 28-79 | **Letterbox-aware video rect calculation duplicated.** Both files compute intrinsic aspect ratio → pillarbox/letterbox insets identically. | Extract to `dom-utils.js` as `getRenderedVideoRect(video, mountTarget)`. |
| `ui/control-integration.js` | 54-59, 205-263 | **Five interrelated boolean flags for visibility state.** `visibilityEnabled`, `panelHovered`, `controlsVisible`, `cursorVisible` + `cursorTimer` create a complex implicit state machine spread across 10+ handlers. State transitions are hard to reason about. | Replace with an explicit state enum (e.g., `'hidden' | 'cursor-only' | 'controls-visible' | 'panel-hovered'`) with a single transition function. |
| `overlay-controller.js` | 95-143, 260-263 | **`getBottomControlLift()` queries all interactive nodes on every call.** Called from `updateLayoutMetrics()` which runs on 250ms interval + ResizeObserver + every render. Each call does `querySelectorAll` + `getBoundingClientRect` for every button in the player. | Cache the result and invalidate on mutation, or throttle the DOM scan. |
| `core/control-keyboard.js` | 19-45 | **Redundant duplicate key bindings.** Both lowercase and uppercase versions mapped (e.g., `d` and `D` both → `toggleDualSub`). Since `_handleKeyDown` doesn't check `shiftKey` for these keys, and `event.key` already provides the case, the uppercase entries are dead weight. The `KeyR` check at line 89 already uses `event.shiftKey` properly — the uppercase key entries in the map are never reached because shift+d produces `D` but the shift check happens after the binding lookup. | Remove uppercase entries or use `event.key.toLowerCase()` in lookup. |

## Triage Response

This section records the current project stance after review against the active control-ownership contract and current code.

### Agree / Prioritize

- `platform/netflix-injected.js`
  - The old `core/auto-pause.js` concern is superseded. Auto-pause is now page-owned, and the old content-script `controllerPausePending` path no longer exists.
  - Do not reintroduce a second pause-owner or a content-script pending-pause flag.
- `platform/netflix-adapter.js`
  - The `MutationObserver` on `document.documentElement` is still broad and undebounced.
  - Keep as a real race/performance risk.
- `core/translation-queue.js`
  - `prefetch()` does sequential `await` work against mutable shared state and can race with `clear()`.
  - Keep as a real state-consistency issue.
- `utils/language-utils.js`
  - `toTranslationKey()` ignoring `title` is a real cache-collision risk.
- `manifest.json` + `utils/extension-api.js`
  - Loading `extension-api.js` in both content-script entries resets the in-memory debug log array.
- `core/control-actions.js`
  - `togglePlayPause()` has an inconsistent return contract.
  - Low severity, but real.

### Disagree

- `content-script.js`
  - The “no teardown on SPA navigation” item is overstated as a current P0.
  - The runtime is intentionally long-lived; playback activation and release now flow through adapter watch-session state and mount rebinding.
  - This can still be revisited, but it is not clearly the top broken path anymore.
- `core/control-actions.js`
  - `seekToTarget()` resume behavior is not inverted under the current product contract.
  - Subtitle navigation is intentionally defined to resume playback after previous / next / repeat.
- `ui/control-integration.js`
  - The current `ensureMounted()` / `render()` split is not considered a proven duplicate-listener race at this time.
  - The attach/detach contract is covered by focused unit tests.

### Deprioritize / Quality Only

- Debug and tracing code size in `content-script.js`
- Duplicate helper implementations (`traceTranslation`, visibility checks, rendered video rect math)
- Keyboard uppercase mapping cleanup
- Hardcoded default model identifiers
- DOM-scan optimizations like `overlay-controller.js` bottom-control lift caching

### Current Priority Order

1. Fix `controllerPausePending` so auto-pause cannot wedge pause ownership.
2. Debounce or narrow adapter DOM scanning.
3. Harden translation-queue async state transitions.
4. Fix translation cache key collisions.
5. Remove duplicate `extension-api.js` load.
   - Treat this as a free cleanup to bundle into the next safe patch, not as a separate high-risk work item.
